SECTION 1: SYSTEM VISION



Build a fully integrated, enterprise-grade school ecosystem where Website, ERP, Mobile App, Blog, Email, and AI are completely connected.

Any action in one system reflects across all systems in real-time.

Single source of truth: the ERP PostgreSQL database.

Everything goes through the /api/v1/ layer. No system talks to another directly.



SECTION 2: ECOSYSTEM ARCHITECTURE



www.bfpsedu.in (Website)

  <--> /api/v1/leads, /contact, /careers, /gallery, /cms, /blog-posts

  <--> ERP Database (via API only)

  <--> Cloudinary (images, gallery)



erp.bfpsedu.in (ERP)

  <--> PostgreSQL on Supabase (primary database)

  <--> Redis on Upstash (cache + rate limiting)

  <--> Serverbyte SMTP (email via noreply@bfpsedu.in)

  <--> Serverbyte IMAP (read leave@, info@, careers@)

  <--> Twilio/Meta API (WhatsApp messages)

  <--> Razorpay (payment gateway)

  <--> Cloudinary (file storage)

  <--> Firebase Admin (push notifications via FCM)

  <--> Sentry (error tracking)



admin.bfpsedu.in (Website Admin Panel)

  <--> /api/v1/cms/* endpoints

  <--> Manages all website content without code changes



blog.bfpsedu.in (Blog)

  <--> /api/v1/blog-posts (read posts)

  <--> /api/v1/blog/auto-publish (automation webhook -- future connection)



Mobile App (Flutter)

  <--> /api/v1/* (all data via REST API)

  <--> Firebase FCM (push notifications)

  <--> Razorpay Flutter SDK (in-app payments)

SECTION 6: BLOG AUTOMATION HOOK (FUTURE)



The blog at blog.bfpsedu.in has an auto-publish webhook ready from day 1.

When external automation tool is connected, it can POST to:

POST /api/v1/blog/auto-publish

Header: X-Blog-Automation-Key: [key from env]

Body: { title, content, category, tags, featuredImage, seoTitle, seoDescription }

Action: Post is created and immediately published on blog.bfpsedu.in

Connection to existing automation tool is a POST-LAUNCH task. Endpoint ready on launch.



SECTION 7: PAYMENT GATEWAY INTEGRATION



Razorpay is integrated from Day 1. Not deferred.

Fee collection: erp.bfpsedu.in -> Razorpay order -> parent pays -> webhook -> DB updated

Mobile app: Razorpay Flutter SDK for in-app payments by parents

Security: Card data never touches BFPS servers. PCI-DSS compliant via Razorpay.

Webhook: Signature verified on every payment callback

Reconciliation: Daily at 2am -- Razorpay vs DB cross-check. Alert on discrepancy.
