SECTION 1: GLOBAL STANDARDS



1.1 Request/Response Format

Base URL: https://api.bfpsedu.in/api/v1/

Content-Type: application/json (all requests)

Auth header: Authorization: Bearer [JWT_ACCESS_TOKEN]



SUCCESS RESPONSE:

{ success: true, data: {...}, message: 'string', pagination?: { page, limit, total, totalPages } }



ERROR RESPONSE:

{ success: false, error: { code: 'ERROR_CODE', message: 'Human readable', details?: [...] } }



1.2 HTTP Status Codes Used

Code

Meaning

When Used

OK

SECTION 2: AUTH ENDPOINTS



POST   /api/v1/auth/login

  Auth: Public

  Body: { email: string, password: string, totpCode?: string }

  Returns: { accessToken, refreshToken, role, user: { id, name, email, photo } }

  Errors: 401 (wrong creds), 423 (account locked), 403 (2FA required)



POST   /api/v1/auth/logout

  Auth: Required

  Body: {} (uses token from header)

  Action: Blacklists access token in Redis, revokes refresh token



POST   /api/v1/auth/refresh-token

  Auth: Public (uses refresh token)

  Body: { refreshToken: string }

  Returns: { accessToken, refreshToken } (rotation -- old token invalidated)



POST   /api/v1/auth/forgot-password

  Auth: Public

  Body: { email: string }

  Action: Sends reset link to email if account exists (no user enumeration)



POST   /api/v1/auth/reset-password

  Auth: Public (uses reset token from email)

  Body: { token: string, newPassword: string }



POST   /api/v1/auth/setup-2fa

  Auth: Required

  Returns: { qrCodeUrl, secret } for Google Authenticator setup



POST   /api/v1/auth/verify-2fa

  Auth: Required

  Body: { totpCode: string }

  Action: Enables 2FA on account after first verification



GET    /api/v1/auth/sessions

  Auth: Required (own sessions only)

  Returns: List of active sessions with device, IP, created time



DELETE /api/v1/auth/sessions/:id

  Auth: Required

  Action: Revoke specific session (logout from one device)



SECTION 3: STUDENT ENDPOINTS



GET    /api/v1/students

  Auth: Required | Roles: All except TELECALLING

  Query: classId, section, stream, search(name/admissionNo), isActive, page(1), limit(50)

  Returns: Paginated student list with class info



GET    /api/v1/students/:id

  Auth: Required | Roles: All (STUDENT/PARENT see only own)

  Returns: Full student profile



GET    /api/v1/students/:id/profile

  Auth: Required

  Returns: Full profile + attendance %, fee outstanding, latest exam results



POST   /api/v1/students

  Auth: Required | Roles: MASTER_ADMIN, ADMIN, PRINCIPAL, VP, RECEPTION

  Body: multipart/form-data { studentData: JSON, photo?: File }

  Returns: Created student object



PUT    /api/v1/students/:id

  Auth: Required | Roles: MASTER_ADMIN, ADMIN, PRINCIPAL, VP, RECEPTION

  Body: Partial student fields to update



DELETE /api/v1/students/:id

  Auth: Required | Roles: MASTER_ADMIN, ADMIN

  Action: Soft delete (isActive = false). Data preserved.



POST   /api/v1/students/bulk-import

  Auth: Required | Roles: MASTER_ADMIN, ADMIN, IT_DEPT

  Body: multipart/form-data { file: CSV|XLSX|TXT|DOCX }

  Returns: { created: N, failed: M, errors: [{row, reason}] }



GET    /api/v1/students/bulk-import/template

  Auth: Required

  Returns: CSV template file download



GET    /api/v1/students/:id/id-card

  Auth: Required | Roles: MASTER_ADMIN, ADMIN, IT_DEPT, RECEPTION

  Returns: PDF student ID card



SECTION 4: ATTENDANCE ENDPOINTS



POST   /api/v1/attendance/mark

  Auth: Required | Roles: TEACHER, AC_COORDINATOR, ADMIN, MASTER_ADMIN

  Body: { classId, date, period(1-8), subjectId, records: [{studentId, status, remarks?}] }

  Validation: Cannot mark future date. Cannot mark same class+date+period twice.

  Action: Creates attendance records. Triggers WhatsApp for absent students.



GET    /api/v1/attendance

  Auth: Required

  Query: classId, studentId, date(YYYY-MM-DD), dateFrom, dateTo, period, status



GET    /api/v1/attendance/summary

  Auth: Required

  Query: classId, month(1-12), year, academicYear

  Returns: Class-wise attendance summary with per-student percentages



GET    /api/v1/attendance/:studentId/percentage

  Auth: Required (STUDENT sees own only)

  Query: month, year, academicYear

  Returns: { percentage, totalDays, presentDays, absentDays, lateDays, isBelowThreshold }



GET    /api/v1/attendance/below-threshold

  Auth: Required | Roles: TEACHER and above

  Query: classId, threshold(default=75), academicYear

  Returns: Students below attendance threshold with their percentage



SECTION 5: FEE ENDPOINTS



GET    /api/v1/fees/structure

  Auth: Required

  Query: classId, academicYear, feeType



POST   /api/v1/fees/structure

  Auth: Required | Roles: MASTER_ADMIN, ADMIN, ACCOUNTS

  Body: { classId, feeType, amount, dueDate, frequency, academicYear }



POST   /api/v1/fees/payment/create-order

  Auth: Required | Roles: ACCOUNTS, RECEPTION, ADMIN, MASTER_ADMIN

  Body: { studentId, feeStructureId, amount, idempotencyKey }

  Returns: { razorpayOrderId, amount, currency, keyId }



POST   /api/v1/fees/payment/verify

  Auth: Required

  Body: { razorpayOrderId, razorpayPaymentId, razorpaySignature }

  Action: Verifies Razorpay signature. Records payment. Generates receipt. Sends email.



POST   /api/v1/fees/payment/webhook

  Auth: Razorpay webhook signature (not JWT)

  Action: Handles Razorpay payment events (paid, failed, refunded)



GET    /api/v1/fees/student/:id

  Auth: Required (STUDENT/PARENT see own only)

  Returns: Complete fee ledger: structure, payments, outstanding balance



GET    /api/v1/fees/defaulters

  Auth: Required | Roles: ACCOUNTS, ADMIN, PRINCIPAL, MASTER_ADMIN

  Query: classId, academicYear, minAmount(default=0)

  Returns: Students with outstanding fees



GET    /api/v1/fees/receipt/:paymentId

  Auth: Required

  Returns: PDF fee receipt



POST   /api/v1/fees/reminders

  Auth: Required | Roles: ACCOUNTS, ADMIN, MASTER_ADMIN

  Body: { classId?, studentIds?, customMessage? }

  Action: Sends WhatsApp reminders to selected defaulters



POST   /api/v1/fees/refund

  Auth: Required | Roles: MASTER_ADMIN, ADMIN

  Body: { paymentId, amount, reason }



GET    /api/v1/fees/reconcile

  Auth: Required | Roles: MASTER_ADMIN, ACCOUNTS

  Action: Cross-checks Razorpay records against DB. Returns discrepancy report.
