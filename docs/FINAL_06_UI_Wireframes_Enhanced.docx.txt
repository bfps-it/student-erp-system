1.3 Component Standards

Buttons: Primary (navy fill), Secondary (blue outline), Danger (red fill), Ghost (transparent)

Cards: white bg, 1px border #E2E8F0, 8px border-radius, 16px padding, box-shadow sm

Tables: alternating gray/white rows, sticky header, sortable columns, pagination

Forms: 48px input height, 8px border-radius, red error states, green success states

Badges: pill shape, color-coded per status (green=active, red=inactive, gold=pending)

Loading: Skeleton loaders (not spinners). Every list and card has skeleton state.

Errors: Toast notifications (top-right, auto-dismiss 5s), with action buttons

Mobile: 100% responsive. Min tap target: 44px. No horizontal scroll.




SECTION 2: ERP SCREENS (erp.bfpsedu.in)



2.1 Login Page

URL: /login

Elements:

  - BFPS logo (centered)

  - Headline: 'BFPS ERP System'

  - Input: Email address

  - Input: Password (show/hide toggle)

  - Button: 'Sign In' (primary, full width)

  - Link: 'Forgot Password?'

  - If 2FA required: shows TOTP input after password

  - Error states: red banner below inputs

  - Lock message: 'Account locked for X minutes' with countdown timer



2.2 Admin Dashboard (/dashboard)

LAYOUT: Sidebar (left, 240px) + Top Navbar + Main Content



SIDEBAR:

  - BFPS logo + school name

  - Navigation items (role-based, grayed out if no access):

    Dashboard | Students | Attendance | Fees | Examinations

    Staff | Leaves | Timetable | Library | Transport | Hostel

    Lead CRM | Notifications | Blog | Website CMS | Gallery

    Careers | Contact Enquiries | Analytics | Reports

    Settings (Master Admin only) | Health Monitor (Admin/IT)

  - Collapse toggle (icon-only mode)

  - User profile at bottom: photo, name, role badge, logout



TOP NAVBAR:

  - Breadcrumb (left): Dashboard

  - Search bar (center): global search across students, staff, leads

  - Notification bell (right): badge count of unread notifications

  - Dark mode toggle

  - User avatar dropdown: Profile | Change Password | Logout



MAIN CONTENT (Admin Dashboard):

  ROW 1 -- KPI Cards (4 cards):

    [Total Students] [Today Attendance %] [Fees Collected Today] [New Leads Today]

    Each card: metric value (large), label, trend arrow vs yesterday



  ROW 2 -- Charts:

    Left (60%): Attendance Trend (LineChart, last 30 days, Recharts)

    Right (40%): Fees Collection (BarChart, monthly, current year)



  ROW 3:

    Left (50%): Recent Activity Feed (last 20 system actions with user, time, action)

    Right (50%): Quick Actions panel

      [Mark Attendance] [Collect Fee] [Add Lead] [Send Notification]

      [View Defaulters] [Leaves Pending] [New Career Applications]



2.3 Students Module

LIST PAGE (/students):

  - Filter bar: Class | Section | Stream | Status(Active/Inactive) | Search

  - Action buttons: [Add Student] [Bulk Import] [Export]

  - Table: Photo | Adm.No | Name | Class | Section | Parent Phone | Status | Actions

  - Row actions: View | Edit | Print ID Card | Delete

  - Pagination: 50 per page with total count



ADD STUDENT DRAWER (slide-in from right):

  Tab 1 - Personal: Name, DOB, Gender, Blood Group, Religion, Category

  Tab 2 - Academic: Class, Section, Roll No, Admission Date, Previous School

  Tab 3 - Family: Parent Name, Mother Name, Phone, Alt Phone, Email, Occupation

  Tab 4 - Address: Full address, City, State, Pincode

  Tab 5 - Documents: Photo upload, Aadhaar (encrypted), TC Number

  Buttons: [Save Draft] [Submit]



STUDENT PROFILE (/students/:id):

  Header: Photo | Name | Admission No | Class | Status badge

  Tabs:

    Overview: Contact info, family, address, academic history

    Attendance: Calendar heatmap + monthly percentage table

    Fees: Ledger table + outstanding balance + Pay button

    Exams: Results table per exam with grades and rank

    Homework: Assigned and submitted homework list

    Documents: Uploaded documents (Aadhaar masked, photo, TC)

    Journey: Academic timeline from admission to now



2.4 Attendance Module

MARK ATTENDANCE (/attendance/mark):

  Step 1: Select Class + Section + Date + Period + Subject

  Step 2: Student list loads -- quick toggle: [P] Present [A] Absent [L] Late

  Step 3: [Mark All Present] shortcut button

  Step 4: [Save Attendance] button -- shows confirmation count

  Already marked: shows existing records (edit mode for corrections)



REPORTS (/attendance/reports):

  - Class-wise monthly report (table + chart)

  - Below 75% alert list

  - Individual student attendance history

  - Export: PDF | Excel | CSV



2.5 Fee Management Module

FEE COLLECTION (/fees/collect):

  Step 1: Search student by name or admission number

  Step 2: Shows: fee structure due, outstanding balance

  Step 3: Select fee type(s) + amount

  Step 4: Select payment method (Cash/Razorpay/Cheque)

  Step 5: [Collect Payment] -> Razorpay opens for online / record for cash

  Step 6: Receipt generated + email sent to parent



DEFAULTER LIST (/fees/defaulters):

  - Filter: Class | Min Outstanding Amount | Academic Year

  - Table: Name | Class | Outstanding | Last Paid | Actions

  - [Send WhatsApp Reminder] per row or [Bulk Remind Selected]

  - Export to Excel



2.6 Leaves Panel (/leaves)

See FINAL_02 Leave Management document for complete UI spec

Summary:

  - Stats: Pending Principal | Pending Director | Approved | Rejected

  - Filter: Status | Source (EMAIL/APP/MERGED) | Date range | Teacher

  - Table: Ticket ID | Teacher | Type | Dates | Days | Source Tag | Status Badge

  - Detail view: Full ticket + approval timeline + action buttons (role-based)



2.7 Lead CRM (/leads)

KANBAN VIEW (default):

  Columns: New | Contacted | Visit Scheduled | Visit Done | Applied | Admitted | Lost

  Cards: Student name | Parent | Phone | Source badge | Date | Assigned to

  Drag-and-drop between columns to update status



TABLE VIEW (toggle):

  All leads in filterable table with search



LEAD DETAIL:

  Student and parent info | Status timeline | Call logs | Notes | WhatsApp button

  [Convert to Admission] button when status = Applied



2.8 Settings (Master Admin Only)

/settings/roles:

  - List of all roles

  - Each role: expandable permission checklist

  - Toggle any permission ON/OFF per role

  - Save changes (takes effect on user's next login)



/settings/users:

  - All users list

  - Add user: name, email, role, temp password

  - Edit role: change assigned role

  - Disable/enable account

  NOTE: superadmin@bfpsedu.in does NOT appear in this list

3.3 Contact Page (/contact)

LEFT HALF: Google Maps embed | Address | Phone numbers | Email

RIGHT HALF: Contact Form

  Fields: Full Name (req) | Phone | Email (req) | Subject (req) | Message (req, min 20 chars)

  Submit: POST /api/v1/contact

  Success state: green banner 'Message received. We reply within 24 hours.'

  Duplicate state: yellow banner 'Enquiry already on file. Our team will contact you.'

  Each card: Position Title | Department | Experience | Description excerpt

APPLICATION FORM:

  Full Name (req) | Phone (req) | Email (req) | Position (dropdown, req)
